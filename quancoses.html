Section 3

HTML

HTML stands for HyperText Markup Language and is used to build the structure of a document.

Elements

Elements are used in HTML to help separate, distinguish, categorize, etc. parts of a webpage.
Syntax for an element: < element type > stuff < / element type >

Examples of Elements

Paragraph Element - This is a block level element, so it will space itself with other elements.
<p> your paragraph... </p>
Division Element - Much like a paragraph element, this is a block-level element allowing for division of sections. This is good for grouping things together. But if another type of block-level element exists, use that instead. A div should really only be used as a last resort.
Span Element - Spans are inline elements. These can also be used for grouping things, such as individual words within a sentence. For example, when applying css to a nhtml document, adding spans within a paragraph can allow you to edit specific words in a sentence.
<div> your content... </div>
h1 through h6 Elements - These are header elements. There can only be a single h1 on a page, and the numerical value must be in ascending order. e.g. h2 must come before an h3, and an h3 must come before an h4.
<h1> your main header </h1>
<h2> your subheader </h2>

HTML Boilerplate

This is standardized markup that needs to be in every html document.

Boilerplate Parts

Doctype - This tells the broswer what type of document it is reading.
<!DOCTYPE html>
HTML Element - This is the top-level element. Everything needs to go in here.
<html lang="en"> all your shit </html>
Title - This is the title of the page which is displayed in the browser tab. Browsers use the title element to create the links in the search results.
<title> your title </title>
Head - This is where all the metadata wil go.
<head> metadata </head>
Body - This is where all the content of the document will go.
<body> touch my body </body>

Lists

Lists can be nested inside one another (both ordered and unordered.)

Unordered Lists

These are lists of things where order is not necessarily needed. This is done by creating a tag named ul with tags named li nested within.
Syntax for unordered lists:
<ul>
<li> stuff </li>.....
< /ul>

Ordered Lists

These are lists where having an order is wanted. This is done by creating a tag named ol with tags named li nested within.
Syntax for ordered lists:
<ol>
<li> stuff </li>.....
< /ol>

Anchor Tags

Anchor tags are typically used for providing a clickable link for navigation to another location. This is done by opening an 'a' tag, having the address specified immedietely after the a, naming the clickable portion between the tags, followed by a closing tag. When adding the location in the href attribute, the link will take you to the location relative to the current directory, if not otherwise specified. So, if we are working in a local directory and you want the link to go to google, make sure to add the http://www.
Syntax for an anchor tag: <a href="location"> stuff </a>

Images

There is only an opening tag for images. Much like the href attribute for an anchor tag, there is an attribute named src for images. Same with the anchor tag, the location of the image used is relative to the current html file being used. Another attribute may be used to help with accessibility, which is the alt attribute. This can be placed right after the src attribute which allows for a text description of the image. This will only show if the image cannot load properly, or if someone is using a screen reader.
Syntax for Images: <img src="img location">
Syntax for Images with alt attribute: <img src="img location" alt="description">

Section 4

Inline Elements

Inline elements only take up the space that they need, so the extra space can be filled with something else.

Examples of inline elements: spans, images, anchor tags

Block Elements

Block elements take up the entire line (block), so even if your content doesn't fill up the whole line, that extra space will still be reserved.

Examples of block elements: divs, paragraphs, h1, h2, ...

More about block elements, inline elements, and inline-block elements will be covered in css.

For example, the 'main' element functions exactly like the generic 'div' element, but it indicates that the content inside is the main. Section Six

CSS: Cascading Style Sheets

Almost everything in CSS follows this basic pattern

selector {
  property: value;
}

It is uncommon, but you can put a style element in the html head section and run css from there. Not so common, but you can do it. Stay away from doing inline styles (styling html using html)
Linking html to css

We link html to css using the link element as follows:

<link rel="stylesheets" href="nameOfYourCss>

Here are all of the css properties listed on MDN

Colors

Different syntax for colors
  • Color Name
  • RGB
    • Syntax: rgb(#, #, #)
    • Ranges from 0 to 255
  • Hexadecimal
    • Syntax example: #FFFC00
    • This works the same way as rgb but instead of going from 0 to 255, we go from 00 to FF. The first two slots are for red, 3rd and 4th are for green, and 5th and 6th are for blue.

Section Seven

Types of Selectors

  • ID - an ID selector is used for styling a single thing. Each ID specific to only 1 thing.
    • #ID name {
        property: value;
      }
  • Class - a class selector is used for multiple things. We can put several things into a class and style them all a certain way.
    • .class name {
        property: value;
      }
  • Descendant - a descendant selector is used to style something that is inside soemthing else. So if we want to style all paragraphs inside of a div a certain way, we would type in 'div space' in place of 'thing thing' below.
    • thing thing {
        property: value;
      }
  • Adjacent - the adjacent selector is used for styling something that comes immedietely after something on the same level. One is not inside (nested) inside of another, but is adjacent to it. Typing in h1 + p in place of thing + thing below would style all paragraphs that are immedietely after h1s.
    • thing + thing {
        property: value;
      }
  • Direct-Descendant - this is a lot like the desdendant selector, but it only applys to a child one level down.
    • thing > thing {
        property: value;
      }
