Section 3

HTML

HTML stands for HyperText Markup Language and is used to build the structure of a document.

Elements

Elements are used in HTML to help separate, distinguish, categorize, etc. parts of a webpage.
Syntax for an element: < element type > stuff < / element type >

Examples of Elements

Paragraph Element - This is a block level element, so it will space itself with other elements.
<p> your paragraph... </p>
Division Element - Much like a paragraph element, this is a block-level element allowing for division of sections. This is good for grouping things together. But if another type of block-level element exists, use that instead. A div should really only be used as a last resort.
Span Element - Spans are inline elements. These can also be used for grouping things, such as individual words within a sentence. For example, when applying css to a nhtml document, adding spans within a paragraph can allow you to edit specific words in a sentence.
<div> your content... </div>
h1 through h6 Elements - These are header elements. There can only be a single h1 on a page, and the numerical value must be in ascending order. e.g. h2 must come before an h3, and an h3 must come before an h4.
<h1> your main header </h1>
<h2> your subheader </h2>

HTML Boilerplate

This is standardized markup that needs to be in every html document.

Boilerplate Parts

Doctype - This tells the broswer what type of document it is reading.
<!DOCTYPE html>
HTML Element - This is the top-level element. Everything needs to go in here.
<html lang="en"> all your shit </html>
Title - This is the title of the page which is displayed in the browser tab. Browsers use the title element to create the links in the search results.
<title> your title </title>
Head - This is where all the metadata wil go.
<head> metadata </head>
Body - This is where all the content of the document will go.
<body> touch my body </body>

Lists

Lists can be nested inside one another (both ordered and unordered.)

Unordered Lists

These are lists of things where order is not necessarily needed. This is done by creating a tag named ul with tags named li nested within.
Syntax for unordered lists:
<ul>
<li> stuff </li>.....
< /ul>

Ordered Lists

These are lists where having an order is wanted. This is done by creating a tag named ol with tags named li nested within.
Syntax for ordered lists:
<ol>
<li> stuff </li>.....
< /ol>

Anchor Tags

Anchor tags are typically used for providing a clickable link for navigation to another location. This is done by opening an 'a' tag, having the address specified immedietely after the a, naming the clickable portion between the tags, followed by a closing tag. When adding the location in the href attribute, the link will take you to the location relative to the current directory, if not otherwise specified. So, if we are working in a local directory and you want the link to go to google, make sure to add the http://www.
Syntax for an anchor tag: <a href="location"> stuff </a>

Images

There is only an opening tag for images. Much like the href attribute for an anchor tag, there is an attribute named src for images. Same with the anchor tag, the location of the image used is relative to the current html file being used. Another attribute may be used to help with accessibility, which is the alt attribute. This can be placed right after the src attribute which allows for a text description of the image. This will only show if the image cannot load properly, or if someone is using a screen reader.
Syntax for Images: <img src="img location">
Syntax for Images with alt attribute: <img src="img location" alt="description">

Section Seven

Types of Selectors

  • ID - an ID selector is used for styling a single thing. Each ID specific to only 1 thing.
    • #ID name {
        property: value;
      }
  • Class - a class selector is used for multiple things. We can put several things into a class and style them all a certain way.
    • .class name {
        property: value;
      }
  • Descendant - a descendant selector is used to style something that is inside soemthing else. So if we want to style all paragraphs inside of a div a certain way, we would type in 'div space' in place of 'thing thing' below.
    • thing thing {
        property: value;
      }
  • Adjacent - the adjacent selector is used for styling something that comes immedietely after something on the same level. One is not inside (nested) inside of another, but is adjacent to it. Typing in h1 + p in place of thing + thing below would style all paragraphs that are immedietely after h1s.
    • thing + thing {
        property: value;
      }
  • Direct-Descendant - this is a lot like the desdendant selector, but it only applys to a child one level down.
    • thing > thing {
        property: value;
      }
  • Attribute - this is a way to select things based off a specific attribute that they have. For example, we might want to style all inputs that have type="text". So, we would put input in place of 'thing' and type='text' in place of 'attribute below.
    • thing[attribute] {
        property: value;
      }

Pseudo Classes

These can be added to the end of a selector to modify an element during a specific state.
Some examples include:
  • thing:hover
  • thing:first
  • thing:first-child
  • thing:active
  • thing:not()
  • thing:nth-child()
  • thing:nth-of-type()
Here is a list of all the pseudo classes

Pseudo Elements

These are like pseduo classes, but instead of styling the state of the element, it styles a specific part of the element: such as the first letter or text area when selected (highlighted)
Some examples include:
  • thing::first-letter
  • thing::first-line
  • thing::selection
Here is a list of all the pseudo elements

Why is it Called Cascade

All this means is that the order we style things matters, assuming the same specificity. So if I made all h2s blue on a css document but then further down the document changed all h2s again to blue, all h2s would then be blue.

Specificity

When multiple selectors are used to style soemthing, the more specific selector will win.
Inline Styles (dont use these) > ID > Class, Attribute, & Pseudo-class Selectors > Element & Pseudo-element selectors
There is also an exception called '!important'. This just always wins, but it should be use sparingly or never. To use !important, add it in as follows:

selector {
  property: value !important;
}

Inheritence

A nested element(child) will inherit the styles of its parent unless otherwise specified. Certain things like buttons dont inherit the styles of their parents (closest parent), so if you want to, you can add the keyword inherit into css. Shown below:

button {
  property: inherit;
}

Percentages are relative to some other value so it is taking a percentage from its parent or the element itself. The height and width property for example take from the parent. Section Ten

CSS Flexbox

What is it?
    Flexbox is a one-dimensional layout method for laying out items in rows or columns.
    Flexbox allows us to distribute space dynamically across elements of an unknown size.

Flex-Direction: This defines what the main axis is and the direction (order). Default is set to row.

Syntax:
Selector {
  Flex-direction: [yourInput];
}
  • Row: The first element in the box will be placed at the left with each element following to the right.
  • Row Reverse: The first element in the box will be placed at the right with each element following to the left.
  • Column:
  • Column Reverse

Justify-Content: This defines how space is distributed between and around the items in the box.

Justify-content: flex-start;
 The items are packed flush to each other toward the edge of the alignment container depending on the flex container's main-start side.
Justify-content: flex-end;
 The items are packed flush to each other toward the edge of the alignment container depending on the flex container's main-end side.
Justify-content: center;
 The items are packed flush to each other toward the center of the alignment container along the main axis.
Justify-content: space-between;
 The items are evenly distributed within the alignment container along the main axis. The spacing between each pair of adjacent items is the same. The first item is flush with the main-start edge, and the last item is flush with the main-end edge.
Justify-content: space-around;
 The items are evenly distributed within the alignment container along the main axis. The spacing between each pair of adjacent items is the same. The empty space before the first and after the last item equals half of the space between each pair of adjacent items.

Here are all the options.

Flex-Wrap: The flex items break into multiple lines. This also chooses the direction of our cross axis.

Wrap

This uses a normal main-axis direction and normal cross-axis direction.
Syntax:
Selector {
  flex-wrap: wrap;
}

Wrap-reverse

This has a normal main-axis direction, but reverses the cross-axis direction.
Syntax:
Selector {
  flex-wrap: wrap-reverse;
}

Align-Items: Controls the alignment on an individual line on the cross-axis. Think of this as the height of all the words on a line in microsoft word. the cross-axis.

align-items: flex-start;
 The cross-start margin edges of the flex items are flushed with the cross-start edge of the line.
align-items: flex-end;
  The cross-end margin edges of the flex items are flushed with the cross-end edge of the line.
align-items: center;
  The flex items' margin boxes are centered within the line on the cross-axis. If the cross-size of an item is larger than the flex container, it will overflow equally in both directions.
align-items: baseline;
  Aligns by the basline of the text within the elements.
More here.

Align-Content: Only works when we have multiple lines (columns/rows). Determines the spacing between the lines. Think of this one as the line spacing in microsoft word.

align-content: space-between;
  Distribute items evenly. The first item is flush with the start, the last is flush with the end
align-content: space-around;
  Distribute items evenly. Items have a half-size space on either end.
More here.

Align-Self: Similar to align-items, but it is for individual items.

align-self: center;
 The flex item's margin box is centered within the line on the cross-axis. If the cross-size of the item is larger than the flex container, it will overflow equally in both directions.

Just look here. Literally the same as align-items.

Flex-Basis: Sets the initial size of an element along the main-axis.

flex-basis: [someSize];
 If we set the flex-basis of something in a row, it will determine the width. If we set the flex-basis to something in a column, it will determine the height.

Flex-grow: This allows the elements to grow to grow into all of the available space, along the main axis (assuming no max-width/max-height is assigned).

flex-grow:[unitlessNumber];
 Imagine there are 2 items in a box. Box1 gets assigned 1 for flex-grow and box2 gets assigned 2 for flex-grow. Box 2 will take up twice as much of the available space when growing. Just the available space; not the total space.

Flex-shrink: Exactly the same as flex-grow, but instead of determining the rate something grows, it is determining how fast it shrinks. A bigger number means it will shrink faster.

flex-shrink: [unitlessNumber];

Flex Shorthand: This is a shorthand for grow, shrink, and basis.
